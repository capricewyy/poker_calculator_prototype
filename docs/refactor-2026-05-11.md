# Prototype Refactor — 2026-05-11

This document describes the refactor that takes the single-file prototype
(`poker-calculator-v2.html`) and reshapes it into a deployable static web app
with integration tests, while preserving the v1 prototype as a non-deployed
reference.

## Goals

1. Promote `poker-calculator-v2.html` from "prototype HTML file in the repo
   root" to "deployable static web app under `app/`."
2. Demote `poker-calculator.html` to a non-deployed reference under `legacy/`.
3. Split v2 along clean module boundaries so future work (testing, a
   framework rewrite, additional features) doesn't have to start from a
   1,200-line single file.
4. Add integration tests covering the main user journeys.
5. Lay the directory out so a release pipeline can be added later with no
   restructure — `app/` is the deploy artifact.

## Non-goals

- No behavior changes to the calculator. Same chip math, same settlement,
  same UI flow, same storage key (`poker_v5`).
- No framework adoption. The product will likely be rewritten in a real
  framework later (see TECHNICAL_DESIGN §8); this refactor stays in vanilla
  JS so it stays simple to delete or port.
- No build step. ES modules + static files only. A bundler can be slotted in
  later without changing source structure.
- No CI / deploy config. The directory layout is deploy-ready; actual
  pipeline wiring is a separate task.

## Module boundaries

### `src/state.js`

- The single `state` object (players, buyins, cashouts, dinners, families,
  chip rate, currency).
- `save()` / `load()` against `localStorage` key `poker_v5`.
- `migrateV3()` and v4 → v5 fall-through, identical to today.

State is exported and shared across modules. The prototype's "one mutable
object, top-down re-render on every change" model carries over — no new
reactivity layer.

### `src/calc/chip-math.js`

Pure, DOM-free, no state-object dependency beyond what's passed in.

- `chipVal(chipMoney, chipCount)`
- `c2m(chips, chipMoney, chipCount)` — chips → money
- `m2c(money, chipMoney, chipCount)` — money → chips
- `toChips(amount, unit, chipMoney, chipCount)`
- `fmtMoney(n, currency)`
- `fmtSigned(n, currency)`

### `src/calc/settlement.js`

Pure, DOM-free. Takes state as input, returns derived data.

- `getDinnerShareFor(dinner, playerId)`
- `calcNets(state)` — per-player net positions with rebalance
- `aggregateForSettlement(rows, families)` — collapses families
- `minimizeTransactions(items)` — greedy two-pointer

### `src/ui/*`

One module per Settle tab + one per Players sub-feature (families). Each
module:

- Imports `state` from `state.js`, calc helpers from `src/calc/`.
- Exports its event handlers and `render*` function.
- Receives a `renderAll` callback from `main.js` for cross-tab refresh.

DOM queries stay scoped to the module's tab; no module reads other tabs'
elements.

### `src/main.js`

- Calls `load()` once.
- Defines `renderAll()` which calls every render function in order.
- Wires inline `onclick=`/`onchange=` handlers by hanging the imported
  functions off `window` (the prototype uses inline handlers — keeping that
  pattern avoids a rewrite of every event binding). This is the *only* spot
  in the code that touches `window`.

## Test approach

**Framework:** Playwright. Tests open the real `app/index.html` (served by
`npx serve` or `http-server`) and exercise the UI.

**Coverage targets — main user journeys:**

1. **Setup flow.** Change chip rate + currency. Verify display updates.
2. **Add players.** Add 4 players; verify list + badges.
3. **Log buy-ins.** Log a chip buy-in + a money rebuy; verify amounts in
   buy-in history.
4. **Add a dinner bill (equal split).** Verify per-person share.
5. **Add a dinner bill (custom split).** Verify totals.
6. **Cash out + settlement.** Cash out every player; verify minimum
   transfers add up to zero and pot rebalances when chips don't tally.
7. **Family settlement.** Two players in a family; verify they don't
   transfer to each other and the family node nets correctly.
8. **Persistence.** Reload page; verify state survives via `localStorage`.
9. **Clear all.** Reset; verify everything empties.

Tests live in `app/tests/integration/`. Each journey is one spec file.

## Deploy story

Anyone deploying this app points their static host at `app/`. That's it.

- GitHub Pages / Netlify / Vercel / Cloudflare Pages: serve `app/`,
  `index.html` is the entry.
- No environment variables. No secrets. No backend.
- Bundling (if ever wanted) becomes a `prebuild` step that emits to
  `app/dist/` — source layout doesn't change.
